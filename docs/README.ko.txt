================================================================================
                            RELEASE NOTES TOOL
================================================================================

원시 개발자 체크인 노트로부터 내부용 DevNotes.txt와 고객 배포용
ReleaseNotes.txt 파일을 자동 생성하는 데스크톱 GUI 도구입니다.

customtkinter로 제작되었으며, Lam Research 비주얼 아이덴티티(네이비 + 그린)에
맞춰 디자인되었습니다.


--------------------------------------------------------------------------------
주요 기능
--------------------------------------------------------------------------------

- 올인원 파이프라인
    DevNotes.txt, 패치 라벨, checkinid.txt, Notes.txt를 입력하면 업데이트된
    DevNotes.txt와 새로 생성된 ReleaseNotes.txt를 한 번에 만들어줍니다.

- 쓰기 전 미리보기
    실행할 때마다 모달 미리보기 창이 뜨며, 3개의 탭(Check-in IDs /
    DevNotes preview / ReleaseNotes preview)에서 파일에 어떤 변경이
    이뤄질지 정확히 확인한 후 진행할 수 있습니다.

- 누락 ID 감지
    checkinid.txt에 적은 ID가 Notes.txt에 없을 경우, 미리보기 화면에
    노란색 경고로 표시되어 커밋 전에 입력을 수정할 수 있습니다.

- 타임스탬프 자동 백업
    커밋할 때마다 원본 수정 전에 DevNotes.YYYYMMDD_HHMMSS.bak.txt
    형식으로 백업을 만들기 때문에, 같은 날 여러 번 실행해도 이전 백업이
    덮어쓰여지지 않습니다.

- 유연한 패치 라벨
    Patch, LabPatch, HomeMade 또는 직접 입력한 사용자 정의 prefix를
    사용할 수 있고, 정수(10) 또는 소수(5.1) 번호 모두 지원합니다.

- 독립 실행 헬퍼 페이지
    한 단계만 필요할 경우 Notes -> For_DevNotes 필터링이나
    DevNotes -> ReleaseNotes 재생성을 별도로 실행할 수 있습니다.

--------------------------------------------------------------------------------
요구 사항
--------------------------------------------------------------------------------

- Python 3.10 이상
- customtkinter

의존성 설치:

    pip install customtkinter


--------------------------------------------------------------------------------
사용 방법
--------------------------------------------------------------------------------

저장소를 클론한 후 실행:

    python ReleaseNotesTool_UI_ctk.py


All-in-One 워크플로우:

    1. 기존 DevNotes.txt 파일을 선택합니다.
    2. 드롭다운에서 패치 타입을 선택하고(Patch, LabPatch, HomeMade, 또는
       직접 입력) 패치 번호를 입력합니다 (예: 10 또는 5.1).
    3. checkinid.txt(포함할 체크인 ID 목록)를 선택합니다.
    4. Notes.txt(원시 개발자 노트)를 선택합니다.
    5. "Run Full Pipeline" 버튼을 클릭합니다.
    6. Preview 다이얼로그에서 결과를 확인하세요. 이상한 부분이 있으면
       "Cancel" 클릭.
    7. "Confirm & Write Files"를 클릭하면 실제로 파일이 작성됩니다.

도구는 다음 작업을 수행합니다:

    - 새 패치 블록(라벨 + 오늘 날짜 + 필터링된 체크인)을 DevNotes.txt의
      "Base Version:" 줄 바로 아래에 삽입합니다.
    - 같은 폴더에 새로운 ReleaseNotes.txt를 생성합니다.
    - 원본 DevNotes.txt의 타임스탬프 백업을 저장합니다.


패치 라벨 예시:

    타입       번호      결과
    -------    ------    -----------
    Patch      10        Patch10
    Patch      5.1       Patch5.1
    LabPatch   3         LabPatch3
    HomeMade   5         HomeMade5
    (사용자)   2         HotFix2


--------------------------------------------------------------------------------
파일 구조
--------------------------------------------------------------------------------

    ReleaseNotesTool/
      ReleaseNotesTool_UI_ctk.py   - GUI 메인 진입점
      full_pipeline.py             - build_preview / commit_preview / run_full_pipeline
      notes_to_for_devnotes.py     - 체크인 ID로 Notes.txt 필터링
      ReleaseNotesCreatorv4.py     - DevNotes.txt를 ReleaseNotes.txt로 변환
      README.md


파이프라인 아키텍처:

미리보기에서 사이드 이펙트가 발생하지 않도록 백엔드는 두 단계로 분리되어
있습니다.

    build_preview(devnotes, patch, checkinids, notes)
        -> PipelinePreview   (순수 / 읽기 전용)

    commit_preview(preview, make_backup=True)
        -> 파일 작성 + 백업 생성

    run_full_pipeline(...)   # 하위 호환용 원샷 래퍼
        = commit_preview(build_preview(...))


--------------------------------------------------------------------------------
입력 파일 형식
--------------------------------------------------------------------------------

checkinid.txt
    N.NNNN 형식의 체크인 버전 번호를 포함하는 텍스트라면 무엇이든 가능합니다.
    이름은 선택 사항이며, 매칭에는 숫자 부분만 사용됩니다.

        alice 0.4091
        bob 0.3968
        0.4260


Notes.txt
    표준 내부 노트 형식. 각 체크인 블록은 "Checkin ID:"로 시작하며 80개
    하이픈 구분선으로 분리됩니다. 도구는 DevNotes에 삽입하기 전 내부 전용
    헤더(Developer:, Timestamp:, Release Notes Needed:,
    [Auto Merge Wizard] 블록 등)를 자동으로 제거합니다.


DevNotes.txt
    "Base Version: ..." 라인으로 시작해야 합니다. 새 패치는 이 라인 바로
    아래에 삽입됩니다.


--------------------------------------------------------------------------------
독립 실행 파일로 빌드하기
--------------------------------------------------------------------------------

앱을 단일 .exe(Windows) 또는 바이너리(macOS/Linux)로 패키징하려면:

    pip install pyinstaller

    python -m PyInstaller \
        --clean \
        --onefile \
        --windowed \
        --collect-all customtkinter \
        --collect-all darkdetect \
        --name ReleaseNotesTool \
        ReleaseNotesTool_UI_ctk.py

결과 파일은 dist/ReleaseNotesTool.exe에 생성됩니다. --collect-all 플래그는
customtkinter의 테마 및 폰트 에셋을 실행 파일에 포함시키기 위해 필요합니다.

참고: PyInstaller 빌드 결과물은 플랫폼별로 다릅니다. Windows 사용자에게
배포하려면 Windows에서 빌드, macOS 사용자에게 배포하려면 macOS에서
빌드해야 합니다.


--------------------------------------------------------------------------------
백업에서 복원하기
--------------------------------------------------------------------------------

백업은 DevNotes.txt와 같은 폴더에 타임스탬프 파일명으로 저장됩니다:

    DevNotes.txt
    DevNotes.20260516_205412.bak.txt   <- 5월 16일 오후 8:54 백업
    DevNotes.20260517_091203.bak.txt   <- 5월 17일 오전 9:12 백업

복원하려면 원하는 백업 파일의 이름을 DevNotes.txt로 수동으로 변경하면
됩니다(기존 파일은 덮어쓰기). "Backup에서 복원" UI 기능은 향후 추가될
예정입니다.
